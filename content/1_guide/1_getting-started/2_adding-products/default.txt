## Create new page

Merx comes with a [product blueprint](reference/blueprints/pages/product). Create a new page in the panel and select *Product* as template. Set a price for your product.

## Page model

To get some convenience functions create a own page model and extend the `Wagnerwagner\Merx\ProductPage` class. You can add your own methods to the product page model later on.

[Kirby docs: Page models](https://getkirby.com/docs/guide/templates/page-models)

<code caption="/site/models/product.php">
```php
<?php

class ProductPage extends Wagnerwagner\Merx\ProductPage
{}
```
</code>

## Template

You are completely free to build your very own template that fits to your product.

You most likely want to display the price of the product and a button to add the product to the cart.

<code caption="/site/templates/product.php">
```php
<?php
/** @var ProductPage $page */
?>
…
<span class="price"><?= $page->price() ?></span>
<form id="product-form" action="<?= url('api/shop/cart') ?>" method="post">
  <input type="hidden" name="page" value="<?= $page->uuid() ?>">
  <input type="number" name="quantity" min="1" value="1">
  <button>Add to cart</button>
</form>
…
```
</code>

## Add product to cart

Submitting the product form will create a POST request to the [Merx API endpoint](/reference/api/routes/shop-cart-post) and responds as JSON. Use JavaScript to interfere the submit event and log the response data to the console.

<code caption="script.js">
```js
const productForm = document.getElementById('product-form');
productForm.addEventListener('submit', async (event) => {
  event.preventDefault();
  const response = await fetch(productForm.action, {
    method: 'post',
    body: new FormData(productForm),
  });
  const json = await response.json();
  console.log(json.data);
});
```
</code>

If you don’t want to use JavaScript continue with [Using the API with PHP](/guide/api/using-api-with-php).
