Text:

There are different options to add Merx to your existing Kirby website. If you are starting from scratch you can use the [Starterkit](#install-the-starterkit).

[Kirby docs: Installing plugins](https://getkirby.com/docs/guide/plugins/installing-plugins)

## Requirements

Merx does not have any special requirements. As long as Kirby 5 is running fine, Merx also will be working.
[System requirements of Kirby](https://getkirby.com/docs/reference/system/requirements)

## Download Merx

[Download Merx](https://github.com/wagnerwagner/merx/archive/main.zip) and unzip. Then put the unzipped folder into the `/site/plugins/` folder of your Kirby project.

## Install and update as Git submodule

[Kirby docs: Install & update via Git](https://getkirby.com/docs/guide/install-guide/git)

```bash
# navigate to the root of your kirby website
cd your-website

# install the submodule in plugins direction
git submodule add https://github.com/wagnerwagner/merx.git site/plugins/merx

# commit the changes
git commit -m "Add Merx as submodule"
```

### Update Merx via Git submodule

To update Merx to the latest version, navigate to the merx plugin directory. 

```bash
cd ./site/plugins/merx

# update to the latest version
git checkout main   # ensure main branch is checked out
git pull            # fetch latests changes and update code
```

## Install and update via Composer

Read more about Composer and Kirby on [getkirby.com](https://getkirby.com/docs/guide/install-guide/composer#using-composer-for-kirby-plugins).

```bash
composer require wagnerwagner/merx
```

### Updating Merx using composer

```bash
composer update wagnerwagner/merx
```

## Install with Kirby CLI

```bash
kirby plugin:install https://github.com/wagnerwagner/merx.git
```

It is ideal for new users to explore many of Merx’s possibilities. If you want to create a simple store fast, this would also be a good option for you.
