## Create checkout page

Merx comes with a [checkout blueprint](/reference/blueprints/pages/checkout). Create a new page in the panel and select Checkout as template. By default the status of the page is *unlisted*. There should be only one checkout page. It must be accessible for everyone, so the status must be *listed* or *unlisted*.

### Create checkout form

Add form fields for the data you need to collect from your customer. For the moment we use *invoice* as payment method.

<code caption="/blueprints/checkout.php">
```php
…
<form id="checkout-form" action="<?= url('api/shop/checkout') ?>" method="post">
  <label>
    Email
    <input type="email" name="email">
  </label>
  <input type="hidden" name="paymentmethod" value="invoice">
  <button>Submit order</button>
</form>
…
```
</code>

## Validate customers data

To make sure you get all the data you need from your customers, Merx uses Kirby’s page validation. You can define the required fields and set validate options for each field you want to validate. 
[Kirby docs: Validation](https://getkirby.com/docs/reference/system/validators)
